Structures

Array

Go 中对线性表的概念区分出数组和切片两种实现，数组在使用过程中表现为长度确定的线性表。
在更具体的数组类型判定上，除了数组元素的数据类型，其表长也会被区分，即 [2]int 和 [4]int 是不同的数组类型。在线性表数据结构选择上，我们一般更青睐切片而非数组，因为到真正写代码时，线性表长度往往是不确定或者变化的。

// cmd/compile/internal/types/type.go

// Array contains Type fields specific to array types.
type Array struct {
	Elem  *Type // element type
	Bound int64 // number of elements; <0 if unknown yet
}

// Slice contains Type fields specific to slice types.
type Slice struct {
	Elem *Type // element type
}

// NewArray returns a new fixed-length array Type.
func NewArray(elem *Type, bound int64) *Type {
	if bound < 0 {
		Fatalf("NewArray: invalid bound %v", bound)
	}
	t := New(TARRAY)
	t.Extra = &Array{Elem: elem, Bound: bound}
	t.SetNotInHeap(elem.NotInHeap())
	return t
}

初始化数组时，可以使用 [...]T 的写法让编译器自己推导数组长度，这保证数组仍与切片相区分。

// cmd/compile/internal/types/type.go

// A Type represents a Go type.
type Type struct {
	// Extra contains extra etype-specific fields.
	// As an optimization, those etype-specific structs which contain exactly
	// one pointer-shaped field are stored as values rather than pointers when possible.
	Extra interface{}

	// Width is the width of this Type in bytes.
	Width int64 // valid if Align > 0

	methods    Fields
	allMethods Fields

	Nod  *Node // canonical OTYPE node
	Orig *Type // original type (type literal or predefined type)

	// Cache of composite types, with this type being the element type.
	Cache struct {
		ptr   *Type // *T, or nil
		slice *Type // []T, or nil
	}

	Sym    *Sym  // symbol containing name, for named types
	Vargen int32 // unique name for OTYPE/ONAME

	Etype EType // kind of type
	Align uint8 // the required alignment of this type, in bytes (0 means Width and Align have not yet been computed)

	flags bitset8
}

// New returns a new Type of the specified kind.
func New(et EType) *Type {
	t := &Type{
		Etype: et,
		Width: BADWIDTH,
	}
	t.Orig = t
	switch t.Etype {
		...
	}
	return t
}

Slice

Go 中的切片是长度不定的数组，允许使用时变动其大小，当分配的内存空间不够时会自动扩容，因而比 Array 更加常用。

// cmd/compile/internal/types/type.go

// Slice contains Type fields specific to slice types.
type Slice struct {
	Elem *Type // element type
}

// NewSlice returns the slice Type with element type elem.
func NewSlice(elem *Type) *Type {
	if t := elem.Cache.slice; t != nil {
		if t.Elem() != elem {
			Fatalf("elem mismatch")
		}
		return t
	}

	t := New(TSLICE)
	t.Extra = Slice{Elem: elem}
	elem.Cache.slice = t
	return t
}

切片的运行时数据结构是 SliceHeader，其中分别记录着数组内存的指针、切片的长度和切片的容量。
即切片被分配了大小为 Cap 的连续内存空间 Data，我们可以访问 Len 长度的空间。

// SliceHeader is the runtime representation of a slice.
// It cannot be used safely or portably and its representation may
// change in a later release.
// Moreover, the Data field is not sufficient to guarantee the data
// it references will not be garbage collected, so programs must keep
// a separate, correctly typed pointer to the underlying data.
type SliceHeader struct {
	Data uintptr
	Len  int
	Cap  int
}

切片对底层数组提供了引用的抽象，运行时修改切片长度，实际就对应数组的扩缩容；扩容后，上层看不到切片的变化，但底层的数组已经改变。
切片被当作函数参数传递时，可以理解为函数内使用着 SliceHeader 的浅拷贝，所以 append 造成的 Data 指针变化并不会反映到调用者的 SliceHeader。

切片的底层数组扩容，实际就是为切片分配更大的内存空间，并把原有的元素拷过去。

  • 当期望的 cap 大小 (对应上面的 cap+3) 大于当前 cap 的两倍，则使用期望的 cap；
  • 否则，当当前 len 小于 1024，则将 cap 翻倍；
  • 否则当前切片满足 len>=1024，让 cap 每次增加 1/4，直到 cap 不小于期望的 cap。
